# מערכת דיווח חיובים ותשלומים — MemsiInterface

**תמוז סחר | גרסה 1.0**

מערכת Desktop לניהול ותחזית תיקוני ומכירות מזוודות עבור רשת סניפים, כולל שליפת נתונים בזמן אמת מ-Priority ERP, תחזיות ביקוש, ומעקב מלאי.

---

## דרישות מערכת

- Python 3.11 / 3.12
- Windows 10 / Windows 11
- חיבור לרשת החברה (לגישה לשרת Priority)
- PostgreSQL — מותקן מקומית או גישה לשרת DB משותף

---

## התקנה

### 1. שכפול הפרויקט

```bash
git clone https://github.com/genadyarony-code/MemsiInterface.git
cd MemsiInterface/priority_interface
```

### 2. סביבה וירטואלית

```bash
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
```

### 3. קובץ .env

העתק את `.env.example` ל-`.env` ומלא את הערכים האמיתיים:

```bash
copy .env.example .env
```

```dotenv
PRIORITY_AUTH_HEADER=Basic <base64-token>
PRIORITY_BASE_URL=https://priority.newcinema.co.il/odata/Priority/tabula.ini/ncinema

DB_HOST=localhost
DB_PORT=5432
DB_NAME=priority_cache
DB_USER=postgres
DB_PASSWORD=<your-password>
```

> **חשוב:** לעולם אל תעלה את קובץ `.env` ל-Git. הוא נמצא ב-`.gitignore`.

### 4. הרצה

```bash
python gui_app.py
```

## מודולים עיקריים

| קובץ | תפקיד |
|------|--------|
| `gui_app.py` | חלון ראשי, לשוניות דוחות ומלאי |
| `forecast_tab.py` | לשונית תחזיות — ARIMA / Prophet / XGBoost |
| `forecast_engine.py` | מנוע התחזיות |
| `fetch_combined.py` | שליפה מ-Priority API עם מטמון |
| `inventory_manager.py` | שליפת מלאי מ-PARTBAL |
| `product_identification.py` | זיהוי קטגוריית מזוודה לפי תיאור |
| `cache_manager.py` | ניהול מטמון SQLite מקומי |
| `db_config.py` | פרמטרי חיבור ל-PostgreSQL |
| `logger.py` | לוגים יומיים ב-`~/.memsi/logs/` |

## לוגים

הלוגים נשמרים אוטומטית ב:

```
C:\Users\<username>\.memsi\logs\memsi.log
```

נשמרים 30 קבצי לוג אחרונים (גלגול יומי). בכל הפעלה נבדקת זמינות DB ו-Priority API — תוצאות מוצגות בשורת הסטטוס התחתית.

---

## לשוניות המערכת

1. **בקשת נתונים** — יצירת דוח חיובים חודשי לכל הלקוחות
2. **דוחות לפי לקוחות** — דוח מרוכז על פני מספר חודשים
3. **דוחות לפי סניף** — פילוח לפי סניפי החברה
4. **מעקב מלאי** — שליפת PARTBAL בזמן אמת מ-Priority
5. **תחקור התנהלות מלאי** — סינון לפי מותג/חומר/גודל
6. **זיהוי מוצרים** — ייצוא/ייבוא CSV לטיפול במוצרים לא מזוהים
7. **תחזיות** — ARIMA / Prophet / XGBoost + תמונת מצב סניף
8. **עדכונים** — עדכון מחירונים ומזוודות חדשות

## תחזוקה

### הוספת לקוח חדש
1. הוסף מספר לקוח ל-`TARGET_CUSTOMERS` ב-`fetch_combined.py`
2. הוסף מיפוי ב-`CUSTOMER_REPAIR_PRICING` וב-`CUSTOMER_REPLACEMENT_PRICING` ב-`pricing_data.py`

### הוספת מוצר חדש
הוסף תיאור לקטגוריה המתאימה ב-`LUGGAGE_IDENTIFICATION` ב-`product_identification.py`

### עדכון מחירים
ערוך את המחירונים ב-`pricing_data.py`

## אבטחה

- כל הסודות (טוקן API, פרטי DB) נמצאים ב-`.env` בלבד ואינם מוכנסים לקוד
- `.env` נמצא ב-`.gitignore` ולעולם לא מועלה ל-Git
- אל תשתף את קובץ `.env` בשום ערוץ תקשורת
