# מערכת דיווח חיובים ותשלומים - Priority Interface

## תיאור המערכת
מערכת לשליפת נתונים מ-Priority, עיבוד חיובים ללקוחות ותשלומים לספקים, וייצוא לאקסל עם גליונות נפרדים.

## קבצים במערכת

### 1. fetch_combined.py
הסקריפט הראשי - מושך נתונים מ-Priority ומייצר קובץ אקסל.

**שימוש:**
```python
python fetch_combined.py
```

**תאריכים:** ניתן לשנות את התאריכים בתחתית הקובץ (כרגע: ספטמבר 2025)

### 2. pricing_data.py
מכיל את כל המחירונים:
- **מחירוני תיקונים ללקוחות** - לפי לקוח ומק"ט
- **מחירוני החלפות ללקוחות** - לפי לקוח וסוג מזוודה
- **מחירוני תשלום לספקים** - תיקונים והחלפות

### 3. product_identification.py
מזהה סוג מזוודה לפי תיאור המוצר (טרולי/בינונית/גדולה/ענקית, קלאסית/מותג/מותג על, רכה/קשיחה)

## מבנה הפלט (combined_output.xlsx)

### גליונות לפי לקוח
כל לקוח מקבל גליון נפרד עם מספר הלקוח כשם הגליון.

**עמודות:**
- תעודה, תאריך, הערה 1 לכתיבה
- מספר לקוח, שם לקוח, שם לקוח קופה
- פרטים (ספקים: שין תיק, הילל)
- סטטוס, לטיפול, סניף
- מקט, תיאור מוצר, כמות, מחיר ליחידה
- זיהוי מזוודה, סוג פעולה (תיקון/החלפה)
- **חיוב ללקוח** - המחיר שאנו מחייבים את הלקוח

### גליון תשלום לספקים
מכיל רק שורות שיש בהן ערך בעמודת "פרטים" (שין תיק והילל).

**עמודה נוספת:**
- **תשלום לספק** - המחיר שאנו משלמים לספק

## בעיות שנפתרו

### 1. בעיית מחירון QAS_LAUFER
**בעיה:** לקוח 360250029 (ואחרים) ממופים ל-'QAS_LAUFER' אבל המחירון היה רק 'QAS' ו-'LAUFER'.
**פתרון:** הוספנו מחירון 'QAS_LAUFER' למילון REPAIR_PRICING.

### 2. בעיית זיהוי מזוודות עם גרש
**בעיה:** מוצרים עם גרש בשם (אינץ') לא זוהו בגלל בעיות תחביר.
**פתרון:** שימוש בגרשיים משולשים (''') במקום גרשיים רגילים.

### 3. ניקוי רווחים
**פתרון:** הפונקציה identify_luggage מנקה רווחים מיותרים לפני השוואה.
